# Impact Filter: Quest

## 1. Selling Yourself

_Before communicating this to others, ensure you are fully committed to the idea._

- **Purpose:** A task dependency tool for agents swarming complex deliverables.
- **Importance:** This is an essential component for my agent orchestration system. My unique domain insights do not exist in any other tool. This could become one of the most valuable and most used software tools in the world.
- **Ideal Outcome:** Quest is extremely easy to use because it has a simple and minimal command set with every command and flag being clear in its intent and obvious in its usage. Agents working on quest tasks have exactly what they need readily at-hand, both to accomplish the task and give updates on progress. Agents decomposing large deliverables and creating tasks can directly express thier intentions with tool calls to quest. They can create dependencies and relationships between tasks without ever getting confused. Quest can display a task graph that makes it easy to verify the correctness of what was captured both for agents and humans.

## 2. Best & Worst Results

_Clarify the stakes by exploring the extremes._

- **Best Result:** My framework is so effective that it gains popularity and notoriety, opening all kind of doors for me. I built one of the first genuinely self improving tools. I unlock a new level of massive productivity for the world.
- **Worst Result:** We get stuck using task tools that are bloated, complex and have too many features and dependencies and that don't align well with the orchestration concepts of the framework. The framework is cumbersome to use and never becomes popular. Other builders with worse ideas thrive while my tool languishes. I miss out on the life changing opportunites that arise from shipping innovation at a pivital moment.

## 3. Success Criteria

_List the specific, measurable results that must be true for this to be a success._

- [ ] Criterion 1: Quest can be installed by a new user and used on a project by typing in 5 or fewer commands on linux, mac & windows
- [ ] Criterion 2: Agents actually use quest when instructed to do so
- [ ] Criterion 3: When agents are using quest, the rate at which they make mistakes with it is lower than 1%
- [ ] Criterion 4: All the changes to a task leave an audit trace of who, what & when
- [ ] Criterion 5: Every command and flag maps directly to a clear and necessary use case
- [ ] Criterion 6: The readme gets a power user (non-developer) up an running with it within 5 minutes.
- [ ] Criterion 7: All the data stored by quest is persisted and easily available
